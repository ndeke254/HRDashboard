# HRDashboard вЂ” Claude Project Guide

## What this project is

An R package (`HRDashboard`) that ships a self-contained Shiny HR Analytics Dashboard. The package generates synthetic attendance/payroll data on first run, stores it in a DuckDB database and a Hive-partitioned Parquet warehouse, then serves a single-page dashboard with KPI cards, charts, and live attendance views.

---

## How to run the app

```r
devtools::load_all()          # load package from source
HRDashboard::run_app()        # launches on http://localhost:3000
```

The app auto-compiles SCSS в†’ `www/main.css` and calls `initialize_app()` in `inst/app/global.R` at startup.

## Data architecture

### DuckDB (`inst/data/base/attendance.duckdb`)
Reference/dimension tables:
- `departments` (id, name)
- `employees` (id, name, dept_id, title, hire_date, exit_date, status, hourly_rate)
- `schedules` (employee_id, monвЂ“fri shift types)
- `holidays` (id, name, month, day, is_fixed)

### Parquet warehouse (`inst/data/warehouse/`)
Hive-partitioned fact table: `year={Y}/month={M}/day={D}/*.parquet`

Fact columns: date, employee_id, dept_id, employee_name, dept_name, is_present, is_on_schedule, scheduled_type, is_late, is_early_leave, hours_worked, gross_daily_pay, regular_pay, overtime_pay, daily_kpi_score

All queries hit Parquet via DuckDB's `read_parquet(..., hive_partitioning = true)`. The internal helper `.p()` in `queries.R` returns the glob expression.

## SCSS system

**Do not edit `www/main.css` directly.** Edit `.scss` files, then recompile:

```r
# from inst/app/ working directory:
sass::sass(
  input  = sass::sass_file("styles/main.scss"),
  output = "www/main.css",
  cache  = NULL
)
```

`global.R` does this automatically on app start via `tryCatch`.

### Color palette (from `colors.aco`)
| Variable | Hex | Role |
|----------|-----|------|
| `$headerBg` / `$navy` | `#0F1C1A` | Header/footer bg, dark surfaces |

## Filter system (server.R)

`current_filters` is an `eventReactive` on `input$apply_filters`. It derives `date_from`/`date_to` from the Year/Month/Day selects:

| Selection | date_from | date_to |
|-----------|-----------|---------|
| Year only | Jan 1 of year | Dec 31 of year |
| Year + Month | First of month | Last of month |
| Year + Month + Day | That day | That day |
| "All" year | 2020-01-01 | Sys.Date() |

The filter list passed to all modules:
```r
list(
  departments  = input$departments,   # character vector or "all"
  employees    = input$employees,     # character vector or "all"
  date_from    = <Date>,
  date_to      = <Date>,
  compare_with = "none" | "prev_year" | "prev_month" | "prev_week"
)
```

## Module pattern

All modules follow the standard `{id}UI` / `{id}Server` pattern. Cross-module state: `shared_chart_metric` (reactiveVal) syncs metric selection between `trends` and `comparisons`.

```r
# server.R
shared_chart_metric <- reactiveVal(NULL)
comparisonServer("comparisons", filters = current_filters, shared_metric = shared_chart_metric)
trendsServer("trends",          filters = current_filters, shared_metric = shared_chart_metric)
```

## Common workflows

**Add a new query:**
1. Add function to `R/queries.R` using `.p()`, `.where()`, `.q()` helpers
2. Export with `#' @export`
3. Run `devtools::document()`

**Change colors:**
1. Edit `inst/app/styles/config/variables.scss` only
2. Recompile CSS (see above)
3. Update matching R colors in `inst/app/global.R` `colors` list

**Add a new KPI metric:**
1. Add entry to `metrics_list` in `inst/app/global.R`
2. Add a new `query_*` function in `R/queries.R`
3. Add a new `kpi_overviewUI` / `kpi_overviewServer` call in `ui.R` / `server.R`

**Regenerate synthetic data:**
Delete `inst/data/` directory and restart the app вЂ” `initialize_app()` recreates everything.
